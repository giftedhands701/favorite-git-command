My Favorite Git Command

Git Commit

Git commit command is the main function for saving changes to local repository safely. Commit
takes a snapshot of the project and treats it as a version.

Simply put,commit is the term used for saving changes.

Note:

  • Git does not add changes to a commit automatically. You have to specify
    which file and changes need to be saved before running the Git commit command.

  • The process of specifying which file and changes need to be saved before running the Git commit
    command is known as staging

  • The commit command does not save changes in remote servers, only in the local repository of Git.

Staging

Simply put, staging is a phase where you decide which changes you want to include in the next commit

  • The Git add command moves changes to the staging area.

  • To add a specific file,i.e example.txt file to the staging area, use the command Git add example.txt

  • To add all changes to the staging area, use the command Git add -A

Committing Changes

Git commit command takes a snapshot representing the staged changes.

  • To commit changes, use the command git commit -m"< message>"

  • The -m flag is a message,describing the type of commit

  • Feel free to run the command git status to know your commit status

Conclussion

The Git committing process requires several steps: moving changes to the staging area and saving them with the commit command.
